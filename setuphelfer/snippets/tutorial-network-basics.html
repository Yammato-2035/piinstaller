Software und Standards

Verwendet u. a.: ping, ss/ip, DNS-Resolver (systemd-resolved oder resolv.conf), OpenSSH zum Testen von Fernzugriff und Ports.

  • OpenSSH
  • ss — Manpage

Voraussetzungen

  • Lauffaehiges System im Heimnetz
  • Zugriff auf Router oder DHCP-Liste
  • Basiskenntnisse Terminal

Schritte

  1. IP, Hostname und Gateway pruefen
  2. Namensaufloesung testen
  3. Erreichbarkeit per Ping/Portcheck pruefen
  4. Dienststatus und Firewall gegentesten

Typische Fehler

  • IP-Konflikte im Heimnetz
  • Falscher DNS-Server
  • Dienst aktiv, aber Port blockiert
