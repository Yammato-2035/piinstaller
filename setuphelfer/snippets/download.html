Raspberry Pi

Empfohlen fuer Musikbox, Bilderrahmen, kleine Heimserver und kompakte Lernprojekte.

  • Pi 4 oder Pi 5 empfohlen
  • 4 GB RAM sinnvoll, mehr ist angenehmer
  • Externer Speicher fuer groessere Projekte einplanen

Auf GitHub Releases das passende Paket fuer deinen Pi waehlen (z. B. .deb oder Installer).

Linux-PC

Besser geeignet fuer NAS, groessere Container-Setups, mehrere Dienste und leistungsintensivere Szenarien.

  • Aktuelles Linux-System

Installationswege

Desktop / Linux-PC

Ideal fuer schnellen Einstieg in Monitoring, Backup und Dienste mit mehr Leistungsreserve.

./start.sh
python3 -m backend.app
npm run tauri:dev

Raspberry Pi

Leichtgewichtiges Setup mit Fokus auf Stabilitaet, Netzwerk und reproduzierbaren Updates.

sudo apt update && sudo apt upgrade -y
sudo systemctl enable piinstaller-backend
sudo systemctl start piinstaller-backend

Sicherheitshinweise

  • Installer-Pakete nur aus vertrauenswuerdigen Quellen laden.
  • Nach Installation direkt Updates und Backup aktivieren.
  • Vor Produktivbetrieb die Dokumentation zur Absicherung pruefen.

FAQ zur Installation

Welches System ist empfohlen?

Fuer Einsteiger Raspberry Pi 4/5 oder ein Linux-PC mit aktueller Debian-/Ubuntu-Basis.

Warum zwei Karten unter „Pakete“?

Die Karten beschreiben typische Szenarien (Pi vs. Desktop/Linux-PC). Die Downloads liegen auf derselben GitHub-Release-Seite; dort das passende Artefakt waehlen.

Was tun bei Verbindungsproblemen?

Erst `health` pruefen, dann Netzwerk-/Firewallpfad und zuletzt Dienste neu starten.

Wo finde ich die naechsten Schritte?

Im Tutorial-Bereich sowie in der Dokumentation mit verlinkten Diagnosewegen.
