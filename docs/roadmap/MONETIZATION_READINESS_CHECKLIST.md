# Monetarisierungs-Readiness (Affiliate & Kommunikation)

Technische Stabilität geht vor Monetarisierung. Diese Liste ergänzt die technische Release-Checkliste.

## Transparenz

- [ ] Öffentliche Statusseite inhaltlich konsistent mit `STATUS_MATRIX.md`
- [ ] GitHub: Roadmap + Testmatrix verlinkt (`WEBSITE_TRANSPARENCY_TEST_MATRIX.md`)

## Affiliate

- [ ] Kennzeichnungspflicht umgesetzt (`docs/monetization/AFFILIATE_DISCLOSURE.md`)
- [ ] Keine ungeprüften Produkte als „Setuphelfer getestet“ (`PRODUCT_TEST_BADGES.md`, `RECOMMENDATION_POLICY.md`)
- [ ] Kein Ranking nach Provision (`AFFILIATE_TRANSPARENCY_TEST_MATRIX.md`)

## Tutorials & Risiko

- [ ] Risikostufen definiert (`docs/website/TUTORIAL_RISK_LEVELS.md`)
- [ ] Backup-Hinweis in risikorelevanten Texten (`docs/website/BACKUP_NOTICE_SNIPPETS.md`)

## Recht (Vorbereitung – keine Rechtsberatung)

- [ ] Impressum, Datenschutz, Haftung, Affiliate-Hinweise als Entwurf geprüft
- [ ] Evidence: `docs/evidence/release-gates/legal_release_gate.md` mit Abnahmestatus
