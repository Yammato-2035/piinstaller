# Developer Dashboard Boundary (Knowledge Base)

Das Developer Dashboard dient ausschließlich internen Entwicklungs- und Governance-Zwecken:

- Gate-/Status-Transparenz
- Evidence-Index und Command-Run-Nachweise
- Roadmap- und Next-Prompt-Entscheidungen

Es ist **kein** Endnutzerfeature und kein Ersatz für Operator-Terminalarbeit.

## Kurzregeln

- Nur allowlist-basierte, sichere Kommandoklassen
- Kein freies Terminal, keine freie Shell, kein sudo-Execute
- Keine gefährlichen Writes (Backup/Restore/USB/Partitionierung)
- Jeder Lauf erzeugt strukturierte Evidence
- Statuswechsel nur evidence-basiert, nicht per UI-Button
