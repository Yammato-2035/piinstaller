# Geführte Einstiegspunkte (keine Monoseite)

Ziel: Nutzer finden **Rolle → Aufgabe → Risiko**, nicht eine endlose Marketing-Landingpage.

## Vorschlag Navigation (logisch)

1. **Start** – Was ist Setuphelfer, aktueller Ampelstatus (Link `PUBLIC_STATUS_PAGE`).  
2. **Sicher anfangen** – Backup anlegen, Verify, Glossar.  
3. **Geräte** – Raspberry Pi / Laptop (jeweils Risiko + Backup-Hinweis).  
4. **Recovery** – Rescue-Stick (Status Gelb/Rot), was geht / was nicht.  
5. **Status & Roadmap** – GitHub, Testmatrix, Evidence.  
6. **Community** – minimal (siehe `COMMUNITY_MINIMAL_STRUCTURE.md`).  
7. **Rechtliches** – Impressum, Datenschutz, Affiliate-Hinweis.

## Content-Regeln

- Jede risikorelevante Seite: mindestens **mittlerer** Backup-Hinweis.  
- Keine „garantiert sicher“-Formulierungen.
