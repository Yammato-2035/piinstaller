# Phase 3 – Hero Section produktnah aufgebaut (setuphelfer.de)

Datum: 2026-03-17
Scope: Nur Hero-Bereich der Startseite, keine freie Layout-Neuerfindung.

## 1) Ziel der Phase

Die Hero Section so schaerfen, dass in wenigen Sekunden klar wird:

- SetupHelfer ist produktnah mit PI-Installer verbunden.
- Es handelt sich um eine echte App-Unterstuetzung.
- Raspberry Pi und Linux stehen im Mittelpunkt.
- Der Einstieg richtet sich an Anfaenger.

## 2) Umgesetzte Anpassungen

Datei: `website/setuphelfer-theme/snippets/index.html`

- Hero-Kicker auf Produktbezug gesetzt:
  - "SetupHelfer + PI-Installer"
- Headline neu formuliert mit klarer App-Wirkung:
  - "Der PI-Installer fuer einen klaren Raspberry-Pi- und Linux-Start"
- Subline von allgemein zu produktbezogen verdichtet:
  - App-Begleitung bei Installation, Systemcheck, Projekten
- Primärer CTA auf Produktstart:
  - "PI-Installer starten" -> `download.html`
- Sekundärer CTA fuer Einsteigerfluss:
  - "Gefuehrter Einstieg" -> `guided-start.html`
- Produktvertrauen durch drei kurze Evidenz-Badges:
  - echte App, Einsteigerfokus, Pi/Linux-Fokus
- Alt-Text der Hero-Grafik praezisiert auf Produktwelt:
  - Tux + Raspberry Pi + Linux-Laptop + Desktop
