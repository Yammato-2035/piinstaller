# Community – Minimalstruktur

Ziel: nicht überladen, aber erreichbar.

## Kanäle (Beispiel)

| Kanal | Zweck | Moderation |
|-------|-------|------------|
| GitHub Discussions | technische Fragen, Evidence | Pflichtlektüre Statusseite |
| Forum (optional) | langfristige Threads | klare Regeln, Anti-Spam |

## Einstiegsregeln (öffentlich sichtbar)

1. Vor Restore-/Disk-Fragen: Backup- und Verify-Status angeben.  
2. Keine Schritt-für-Schritt-Anleitungen ohne Risikostufe.  
3. Hardware nur mit Testbadge laut `PRODUCT_TEST_BADGES.md`.

## „Nicht leer“ ohne Chaos

- Pinned: **Status**, **Sicherheitshinweis**, **wo Bugs melden**.  
- Eine Sammelkategorie „Allgemein“, keine Dutzend leerer Unterforen.
