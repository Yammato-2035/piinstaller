# Empfehlungsrichtlinie

1. **Wahrheitsgemäß:** Nur Fähigkeiten beschreiben, die das Produkt wirklich hat (unterstützt, prüft, führt aus – nicht „garantiert“).  
2. **Getrennt:** getestete vs. ungetestete Hardware klar trennen.  
3. **Kein Druck:** keine Dark Patterns, keine falsche Dringlichkeit.  
4. **Backup zuerst:** risikorelevante Empfehlungen verlinken auf Backup-Hinweise.  
5. **Updates:** veraltete Empfehlungen entfernen oder auf „ungeprüft“ zurückstufen.

## Konfliktfall

Wenn technische Stabilität (Rot/Gelb) und Monetarisierung kollidieren: **Technik hat Vorrang.**
