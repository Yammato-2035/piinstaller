# Raspberry-Pi-Projektseiten – Struktur

Jede Projektseite (Beispiel-Vorlage):

1. **Kurzbeschreibung** – was der Nutzer erreicht.  
2. **Voraussetzungen** – Hardware, OS-Version.  
3. **Risikostufe** + Backup-Hinweis (Snippet).  
4. **Schritte** – nummeriert, mit Verifikation.  
5. **Was Setuphelfer tut** – unterstützt / prüft / führt aus (keine Garantien).  
6. **Getestete Konfiguration** – Badge + Link Evidence/Matrix.  
7. **Affiliate-Block** – klar getrennt, mit Kennzeichnung.  
8. **Weiterführend** – Statusseite, GitHub, FAQ.

Keine „copy-paste ohne Risiko“-Sprache bei Boot/Partitionierung.
