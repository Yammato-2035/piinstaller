# Feature-Gatekeeping

## Ziel
Verhindern, dass unfertige Features durch neue ersetzt werden.

---

## REGEL

Neue Features dürfen nur begonnen werden, wenn:

- bestehende Features getestet sind
- dokumentiert sind
- stabil laufen

---

## AKTUELLER STATUS

Backup/Restore:
→ MUSS zuerst abgeschlossen werden

---

## VERBOTEN

- Bluetooth-Integration starten
- neue Module beginnen

solange Backup/Restore nicht freigegeben ist

---

## FREIGABEKRITERIUM

Backup/Restore ist fertig wenn:

- Real-Test erfolgreich
- Sicherheitsprüfung vollständig
- UI integriert
- Dokumentation vollständig
